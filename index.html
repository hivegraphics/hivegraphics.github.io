C++ Primer-基础知识

2015-07-19
学习笔记
1. 初始化和赋值

初始化不是赋值，初始化的含义是创建变量时给其赋予一初始值，而赋值的含义是把对象的当前值擦出，然后再用一个新值替换。

2. 使用列表初始化为变量赋初值

使用列表初始化可以防止无意间错误的给一变量赋初值，编译器会主动提示此错误。这种错误往往发生在不经意之间，所以尽量使用列表初始化为一变量赋初值。

3. 建议初始化每一个内置类型的变量

程序中，我们尽量显式的给每一内置类型变量赋予初始值，防止变量因为未赋初值而变成未定义的类型，此时再试图拷贝或者其它形式来访问该变量将引发程序错误。使用未初始化的变量是一种错误的编程行为。

4. 在第一次使用对象的地方附近定义它可以让我们更容易找到变量的定义，再者就是我们可以通过它的定义给它一个合理的初值。
5. 初始化所有指针

使用未经初始化的指针是引发运行时错误的一大原因，在C++11标准下，建议使用nullptr初始化指针。

6. 面对一条比较复杂指针或引用的声明语句时，从右往左阅读可以很清楚的理解其具体含义

7. 声明和定义

为了支持分离式编译（将源文件分割成若干文件，每个文件可以独立编译），C++将声明和定义区分开来。声明（declaration）使得名字为程序所知，一个文件如果想使用别处定义的名字则必须包含对那个名字的声明。而定义（definition）则负责创建与名字关联的实体。它们有以下两点区别：

  • 声明和定义都规定了变量的类型和名字，但定义还必须为变量申请存储空间
  • 变量只能被定义一次，但可以被多次声明。
8. string对象和字面值拼接，必须保证加法运算符（+）的两侧至少有一个是string

因为某些历史原因，同时也为了兼容C，所以C++中的字符串字面值并不是与标准库类型string对象。在字符串字面值与string类型拼接时，必须确保每个加法运算符的两侧的运算对象至少有一个是string

C++ Primer-面向对象的程序设计

2015-07-09
学习笔记

面向对象的程序设计（Object-oriented programming）的核心思想是数据抽象、继承（Inheritance）和动态绑定（Dynamic binding）。通过使用数据抽象，我们可以将类的接口与实现分离；使用继承，可以定义相似的类型并对其相似关系建模；使用动态绑定，可以在一定程度上忽略相似类型的区别，从而以统一的方式使用它们的对象。

Note1. 派生类构造函数应显式的调用基类构造函数来完成整体初始化

每个类控制它自己的成员初始化过程，尽管派生类对象中含有从基类继承而来的成员，但是派生类并不能直接初始化这些成员。和其他创建基类的代码一样，派生类也必须使用基类的构造函数来初始化它的基类部分。

尽管从语法上的角度我们可以在派生类的构造函数体中给它的公有（Public）或受保护（Protected）成员直接赋值，这样做法是不明智的。每个类负责定义自己的接口，要想与类的对象进行交互就应该使用该类提供的接口，因此在派生类通过构造函数初始化对象的时候，建议调用基类的构造函数进行初始化。

Note2. 使用override说明符说明派生类中的虚函数

在派生类中如果定义了一个函数与基类中的虚函数的名字相同但是形参列表不同，这是合法的行为，编译器会认为这是一个独立的新函数。但这样可能会违背了我们原本的想法，因为我们本来希望派生类覆盖掉基类的版本，但是一不小心把参数列表弄错了。

编程过程中想要调试这样的错误是非常困难的，所以在C++11新标准中建议使用override说明符说明派生类中需要覆盖基类版本的虚函数，此时编译器就会自动的为我们检查覆盖的虚函数是否有误。

Note3. 公有继承和私有继承

类对其继承来的成员访问权限受到两个因素影响：

  1. 基类中该成员的访问说明符
  2. 派生类的派生列表中的访问说明符

公有继承和私有继承制只控制派生类用户对基类成员的访问权限，但对于派生类开发者来说只受到基类对应成员的访问说明符的影响（即：如果基类成员是公有的或受保护的，开发者不管是公有继承或是私有继承都能正常访问；但如果基类成员是私有的，不管是私有继承还是公有继承都不能访问到该私有成员）

struct和class唯一的区别只有默认访问权限的不同（struct默认public，class默认private），就没有其他任何更深层次的不同了。

C++ Primer-类的基本特性

2015-07-07
学习笔记

前段时间一直忙于一些琐碎的事情，直到最近才终于有时间静下心来看看书，于是趁这段时间详读了一遍C++ Primer（第5版）,同时将书中很多比较好的内容整理在博客当中，便于翻阅记忆。

C++ Primer（第5版）是一本很好的学习C++基础的书籍。它不仅完整的介绍了C++的基础知识，同时也介绍了很多最新的C++11标准。其最有价值的地方在于书中有很多小的Note,这些Note提醒开发者需要注意的重点部分，常见的开发陷阱以及良好的程序设计习惯。下面是整理出的一些重要的Note:

Note1.类的隐式this指针

每个类中都包含一个隐式的this指针，指向对象本身。this是一个常量指针，不能改变this指针中保存的地址。

默认的this指针类型是指向类类型的非常量版本的常量指针，通过向类的成员函数参数列表后面添加const关键字，就可以将隐式this指针类型变成指向常量的常量指针，像这样使用的成员函数被称作常量成员函数（const member function）。这样就可以让常量对象能够调用该函数了（常量对象不能调用普通成员函数）。

Note2. 为每个类显式的定义构造函数

如果没有为类提供显式的构造函数，编译器会为这个类提供合成的默认构造函数（synthesized default constructor）。但我们不能依赖于编译器提供的默认构造函数去构造一个类，这样是不明智的选择，原因有三：

  • 编译器只有在我们没有为类提供任何构造函数的情况下，才会生成一个合成的默认构造函数。如果我们显式的提供了一个带参数的构造函数，这个时候这个类就没有默认构造函数了。
  • 对于某些类来说，合成的默认构造函数可能执行错误的操作，用户在创建类的时候可能得到一些未定义的值
  • 编译器不能为某些类提供合成的默认构造函数。例如，如果类中包含一个其他类类型的成员且这个类型没有默认构造函数，那么编译器将无法初始化该成员。

Note3. 封装的益处

封装（encapsulation）的意思是隐藏类的具体实现细节,分离类的接口和实现。封装有两个重要优点：

1. 确保用户代码不会无意间破坏封装对象的状态；
2. 被封装的类的具体实现细节可以随时改变，而无需调整用户级别的代码

Note4. 尽量通过构造函数初始值列表显式的初始化成员

通过构造函数初始值列表显示的初始化成员，可以在执行类的构造函数之前直接初始化该成员。

Markdown语法简介

2015-06-17
技术归档

Markdown是一种轻量级的标记语言，它使用简易的纯文本格式编写文档，然后再转化成有效的HTML文档。我在第一次使用Markdown时便被她的简洁深深的吸引住了，她让我能够更专注的去编写文档的内容而不需要花太多时间去注意文档的格式。

本文通过整理网上各种资料，总结了Markdown的一些基本语法，用作参考文档。同时，推荐一款在线Markdown编辑网站—作业部落，它提供了很多方便实用的功能和强大的实时预览功能。

1. 斜体和粗体

使用 * 和 ** 表示斜体和粗体。

*斜体*
**粗体**

效果：斜体，粗体

2. 超链接

使用 [描述](链接地址) 为文字增加超链接。

点击[本人博客](http://leechengpeng.com/)可以跳转到我的博客。

效果：点击本人博客可以跳转到我的博客。

3. 分级标题

Markdown有两种类型的标题格式：Setext和atx

3.1 Setext

在文本下一行添加一行”=”表示H1
在文本下一行添加一行”-“表示H2

this is an H1
=============
this is an H2
-------------

效果如下：

this is an H1

this is an H2

3.2 atx

在行首插入1到6个“#”，表示标题的级数

效果如下：

H1

H2

H3

H4

H5
H6

4. 列表

4.1 无序列表

使用 *，+，- 表示无序列表（注：*，+，-与文本之间需添加一个空格）。

效果：

  • Red
  • Green
  • Blue

4.2 有序列表

使用数字和点表示有序列表。

效果：

  1. 有序列表项1
  2. 有序列表项2
  3. 有序列表项3

在空行中输入三个或者更多的 “-“, “*”, “_”即可出现一条水平分割线

使用 > 表示文字引用。

7.1 行内代码块

使用形如 `代码` 表示行内代码块。

让我们聊聊 `html`
